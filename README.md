# Shree Swastik Management System (SSMS)

A robust Manufacturing Resource Planning (MRP) and Phase Management system built with React, Tailwind CSS v4, Node.js, Express, and Prisma ORM.

## Running the Project locally

### 1. Start the Backend
Open a terminal and run the backend script:
```bash
cd e:\Swastik\backend
npx nodemon src/index.ts
```

**Default Credentials for testing:**

### 2. Start the Frontend Application
Open another terminal:
```bash
cd e:\Swastik\frontend
npm run dev
```
Access the dashboard at `http://localhost:5173`.

## Roles & Access Control Setup
The system employs strict RBAC built dynamically into React Router (`<ProtectedRoute roles={[...]} />`) and Express Middlewares (`authorizeRoles(...)`):
- **SUPER_ADMIN / ADMIN**: Can accept Phases, schedule queues, override settings.
- **SALES**: Can only insert Leads via Intake form.
- **OPERATOR**: Restricted to Dashboards and reading schedules.

## Core Workflows
1. **Intake**: A Sales rep submits a Lead in `Project Intake` with Dimensions and Initial Budget.
2. **Phase Strict Sequencing**: Using the Phase Manager, an Admin adds Phases (N phases). Phase 2's start button is locked entirely until Phase 1 is marked `ACCEPTED`.
3. **Queue / FullCalendar Scheduler**: Shows visual blockage. Time blocking enforces queuing logically depending on machine availability.
4. **Finance Dashboard**: Real-time budget variance utilizing Recharts to map out Allocated Budget vs. Spent.

## Deployment Instructions
1. Setup a standard host (e.g. VPS running purely on CPU).
2. Change the Database provider in `backend/prisma/schema.prisma` to `postgresql` and place your deployment string in `.env`.
3. Change `baseURL` in `frontend/src/lib/api.ts` to your reverse-proxy target domain (e.g. `https://api.shreeswastik.com`).
4. Build the frontend (`cd frontend && npm run build`) and serve `dist/` with Nginx.
5. Launch backend via PM2 (`cmd: pm2 start dist/index.js`).
